## 遊戲概述

### 背景故事

古老的雷霆之石（Thunderstone）沉睡在深地下城數世紀之久，但黑暗勢力已開始蠢蠢欲動。村莊面臨來自地牢深處怪物的威脅，英雄們必須透過強化自身、購買裝備和法術，逐漸變強以對抗越來越強大的敵人。最終，玩家必須擊敗持有雷霆之石的最強敵人，拯救村莊。

### 遊戲類型特徵

**甲板構築（Deck Building）** 是雷霆之石的核心機制：

- 玩家從基礎卡牌開始
- 通過戰鬥獲得金幣和經驗
- 用金幣購買更強大的卡牌加入自己的牌組
- 隨著牌組升級，戰鬥力逐漸增強
- 最終贏得最終戰鬥

### 差異化亮點

相比 Dominion（甲板構築遊戲的鼻祖），Thunderstone 加入了：

| 特色 | 說明 |
|------|------|
| **戰鬥主題** | 不是抽象的積分，而是真實的戰鬥系統 |
| **英雄升級** | 英雄有 3 個等級，升級後變得更強大 |
| **地牢深度** | 怪物位置決定難度（Rank 1-3） |
| **光源機制** | 光源值影響對特定怪物的傷害 |
| **合作模式** | 支持單人與怪物 AI 對抗 |

## 核心設計理念

### 設計支柱

1. **進度感（Progression）**
   - 玩家需要看到明確的變強過程
   - 英雄升級、牌組增強提供視覺反饋
   - 每場遊戲的難度遞增

2. **策略深度（Strategic Depth）**
   - 需要決策：何時購物？何時進入地牢？
   - 牌組構築有多條路線通往成功
   - 不同英雄組合產生不同遊戲體驗

3. **戲劇張力（Dramatic Tension）**
   - 面對強大怪物時的挑戰感
   - 關鍵決策的緊迫性
   - 最後一戰的史詩性

4. **可重玩性（Replayability）**
   - 英雄/村莊卡牌隨機選擇
   - 地牢牌堆的隨機性
   - 單人模式提供持續的挑戰

### 遊戲平衡原則

| 原則 | 實現方式 |
|------|---------|
| **運氣與技巧的平衡** | 隨機性來自牌組抽取和地牢翻牌，但購買決策完全由玩家控制 |
| **強度之間的平衡** | 高成本卡牌效果更強，但不會絕對壓倒性 |
| **英雄之間的平衡** | 不同英雄專長不同，無絕對最優選擇 |
| **回合長度的平衡** | 單個回合不會太長，遊戲節奏快速 |

## 遊戲目標

### 主要目標

**擊敗持有雷霆之石的怪物，同時保護村莊免受侵害。**

### 勝利條件（多人模式）

玩家需要在遊戲結束時滿足：

```
玩家獲得的卡牌勝利點 (VP) > 逃脫到村莊的怪物勝利點總和
```

**且玩家必須成功獲得雷霆之石**

### 失敗條件

- 雷霆之石在未被任何玩家控制的情況下離開地牢（被推出到村莊外）
- 所有玩家都無法阻止這一發生

### 單人模式目標

與上述相同，但玩家與自動怪物推進系統對抗

## 遊戲設置（Setup）

### 步驟 1：選擇英雄和村莊卡

**對於 2 人遊戲：**
1. 從 10 個英雄系列中隨機選擇 4 個
2. 每個系列有 3 個等級，玩家各自選 1 個系列
3. 從 8 種村莊卡類型中隨機選擇 6 種（去掉 2 種）

**對於 3-4 人遊戲：**
1. 從 10 個英雄系列中隨機選擇 8 個
2. 每位玩家任選 1-2 個系列的起始英雄
3. 使用全部 8 種村莊卡類型

**對於 5 人遊戲：**
1. 使用全部 10 個英雄系列
2. 每位玩家任選 1-2 個系列
3. 使用全部 8 種村莊卡類型，並增加每種的數量

### 步驟 2：設置地牢牌堆（Dungeon Deck）

1. **分級準備**：
   - 將所有怪物卡按等級分為 1 級、2 級、3 級
   - 目標怪物（等級）數量：
     - 1 級怪物：20 張
     - 2 級怪物：20 張
     - 3 級怪物：20 張
     - 總計：60 張（混合 8 種類型）

2. **雷霆之石放置**：
   - 將 **雷霆之石** 卡放在 1 級怪物堆中隨機位置
   - 將 1 級 + 雷霆之石 混合，放在整個地牢牌堆的底部 10 張內

3. **洗牌**：
   - 所有怪物卡混合洗牌
   - 形成 60-70 張的地牢牌堆

### 步驟 3：設置地牢大廳（Dungeon Hall）

1. 從地牢牌堆頂部翻出 6 張怪物卡
2. 排列成 **Rank 3（最遠）→ Rank 1（最近村莊）** 的順序
3. Rank 1 的怪物對所有玩家最具威脅性

```
地牢大廳布局：

[牌堆]
  ▼
[Rank 3] [Rank 2] [Rank 1] → (通向村莊)
```

### 步驟 4：設置村莊商人（Village Market）

1. 從選定的 6-8 種村莊卡中各取 8-10 張
2. 排列成一列，供玩家購買

### 步驟 5：每位玩家的初始牌組

每位玩家從以下基本卡開始：

| 卡牌 | 數量 | 說明 |
|------|------|------|
| 民兵（Militia） | 6 | 攻擊力 +0，基本單位 |
| 火把（Torch） | 4 | 提供 +1 光源值 |
| 匕首（Dagger） | 4 | 攻擊力 +1 |
| 應急乾糧（Rations） | 4 | 戰鬥時 Force +2 |

**初始手牌**：洗牌並抽 5 張牌

### 步驟 6：選擇起始英雄

每位玩家選擇所選英雄系列的 **1 級英雄**，放在遊戲區域（不在手牌中，常駐）

### 遊戲準備完成

所有玩家現在準備好開始第一回合

## 遊戲規則

### 回合結構

每個回合包括三個主要階段：

```
┌─────────────────────────────────────┐
│ 1. 行動階段（Action Phase）         │
│    └─ 決定是否進地牢或休養          │
├─────────────────────────────────────┤
│ 2. 購買階段（Purchase Phase）       │
│    └─ 用金幣購買村莊卡牌           │
├─────────────────────────────────────┤
│ 3. 整理階段（Cleanup Phase）        │
│    └─ 棄棄卡牌，抽新卡牌           │
└─────────────────────────────────────┘
```

---

### 階段 1：行動階段（Action Phase）

#### 1.1 玩家決策

在這個階段，玩家必須選擇**以下兩種選項之一**：

```
╔═══════════════════════════════════════════════════╗
║ 選項 A：進入地牢（Enter the Dungeon）            ║
║ ├─ 檢視 Rank 1 的怪物                          ║
║ ├─ 組織英雄和卡牌進行戰鬥                      ║
║ └─ 戰鬥結果決定獲得或損失                      ║
╠═══════════════════════════════════════════════════╣
║ 選項 B：休養或造訪村莊（Rest/Visit Village）    ║
║ ├─ 不進行戰鬥                                  ║
║ └─ 但怪物會推進一格，可能逃脫到村莊           ║
╚═══════════════════════════════════════════════════╝
```

#### 1.2 選項 A：進入地牢詳細流程

**步驟 A1：選擇目標怪物**
- 玩家查看 Rank 1-3 的所有可見怪物
- 選擇其中一個作為攻擊目標
- 該怪物必須在地牢大廳中可見

**步驟 A2：組織進攻隊伍**

玩家從手牌中選擇以下卡牌**發揮其效果**：

| 卡牌類型 | 效果 | 數量限制 |
|---------|------|--------|
| 英雄 | 提供 Attack 或 Magic Attack | 1 個 |
| 武器 | 增加 +Attack | 多個 |
| 法術 | 增加 +Magic Attack | 多個 |
| 食物 | 增加 Force（生命值上限） | 多個 |
| 光源 | 增加 Light 值，對付光敏怪物 | 多個 |

**步驟 A3：計算總攻擊力**

```
總攻擊力 = 英雄攻擊 + 武器攻擊 + 法術攻擊 + 食物 Force 加成
```

**定義**：
- **Attack（物理攻擊）**：來自英雄、武器
- **Magic Attack（魔法攻擊）**：來自英雄、法術、某些卡牌
- **Light（光源）**：減少「光源懲罰」，某些怪物需要光源才能傷害

**步驟 A4：應用怪物特殊能力**

檢查目標怪物是否有特殊要求或效果：

| 怪物屬性 | 效果 |
|---------|------|
| **光源懲罰** | 如果 Light < 怪物 Light 值，Attack 減少 |
| **魔法免疫** | 只能用 Physical Attack 傷害 |
| **英雄數量要求** | 必須有最少 N 個英雄（怪物指定） |
| **特殊傷害** | 某些傷害類型對該怪物無效 |

**步驟 A5：計算最終傷害**

```
最終傷害 = 總攻擊力 - 怪物防禦修正
```

**步驟 A6：比較傷害與怪物 HP**

- **如果 最終傷害 ≥ 怪物 HP**：
  - ✅ **戰鬥成功** → 玩家獲得怪物
  - 怪物卡轉移到玩家的「獲得堆」
  
- **如果 最終傷害 < 怪物 HP**：
  - ❌ **戰鬥失敗** → 怪物逃脫
  - 怪物卡進入「村莊破壞堆」
  - 地牢大廳留下空位（後續填補）

#### 1.3 選項 B：休養/造訪村莊

**效果**：

1. 玩家**不進行任何戰鬥**
2. 所有地牢中的怪物**向前推進 1 格**：
   - Rank 1 的怪物逃脫到村莊
   - Rank 2 → Rank 1
   - Rank 3 → Rank 2
3. 從地牢牌堆補充新怪物到 Rank 3

**逃脫怪物的影響**：
- 逃脫的怪物進入「村莊破壞堆」（見特殊機制章節）
- 這些怪物的 VP 值在遊戲結束時被扣分

---

### 階段 2：購買階段（Purchase Phase）

#### 2.1 計算可用金幣

玩家從手牌中**仍未使用的卡牌**中獲取金幣：

```
可用金幣 = Σ(所有卡牌的 Gold 值 - 已使用的卡牌 Gold 值)
```

#### 2.2 購買流程

1. **查看村莊市場**：玩家可以購買任何可見的卡牌
2. **支付金幣**：消耗相應金幣購買卡牌
3. **放入「獲得堆」**：購買的卡牌**不立即加入手牌**，而是放入獲得堆

#### 2.3 購買限制

- **無單次購買數量限制**：只要有足夠金幣，可以購買多張卡牌
- **無單張購買限制**：市場中同一張卡牌可以購買多張（數量允許的情況下）
- **金幣消耗完為止**：可以自由決定何時停止購買

#### 2.4 獲得堆機制

所有本回合購買和戰鬥獲得的卡牌進入「獲得堆」，在整理階段時才加入牌組。

---

### 階段 3：整理階段（Cleanup Phase）

#### 3.1 棄棄手牌

- 玩家將**手牌中的所有卡牌棄棄**（放入棄牌堆）
- 已發揮效果的卡牌都被棄棄，包括英雄卡（英雄本身不棄棄，但其卡牌版本棄棄）

**特例**：常駐英雄不被棄棄，始終在遊戲區域

#### 3.2 整合獲得堆

- 本回合所有「獲得的怪物卡」和「購買的村莊卡」都**加入牌組**
- 這些卡牌**立即可用**，下回合可能被抽到

#### 3.3 洗牌與抽取

1. **檢查牌組**：
   - 如果牌組有卡牌，洗牌
   - 如果牌組空（罕見），從棄牌堆重新洗牌成牌組

2. **抽取新手牌**：
   - 從牌組頂部抽 5 張牌
   - 如果牌組卡牌不足 5 張，盡量抽取，再從棄牌堆補充

#### 3.4 回合完全結束

準備下一回合

## 戰鬥系統詳解

### 戰鬥的三大要素

| 要素 | 定義 | 來源 |
|------|------|------|
| **攻擊力（Attack）** | 物理傷害 | 英雄、武器 |
| **魔法攻擊（Magic Attack）** | 魔法傷害 | 英雄、法術 |
| **光源（Light）** | 對付暗系怪物的輔助 | 光源物品 |

### 攻擊計算公式

```
總傷害 = Attack + Magic Attack
最終傷害 = 總傷害 - 光源懲罰調整 + 特殊加成
```

### 光源懲罰系統

**概念**：某些怪物在黑暗中難以被擊敗。

| 情境 | 計算 |
|------|------|
| 怪物有「Light +X」屬性 | 需要 Light 值 ≥ 怪物 Light 值 |
| 玩家 Light < 怪物 Light | 傷害每差 1 點 Light，Attack -1 |
| 玩家 Light ≥ 怪物 Light | 無懲罰，傷害正常 |

**範例**：
```
怪物：白龍（Light +2，HP 7）
玩家：
  - 英雄 Attack +2
  - 武器 Attack +2
  - 光源 Light +1（不足）
  
計算：
  總傷害 = 2 + 2 = 4
  光源差距 = 2 - 1 = 1
  懲罰 = -1
  最終傷害 = 4 - 1 = 3
  結果：3 < 7（HP），戰鬥失敗！
```

### 英雄升級的戰鬥優勢

| 等級 | 攻擊力 | 額外效果 |
|------|--------|--------|
| 1 級 | 基礎 | — |
| 2 級 | 基礎 +1 | 可能有卡牌抽取效果 |
| 3 級 | 基礎 +2 | 更強額外效果，可購買特定卡牌 |

**升級的戰略價值**：
- 投資升級英雄，長期提升戰鬥力
- 英雄升級後，可進地城時抽牌（優勢迴圈）
- 終局戰鬥時，高等級英雄是必需品

## 英雄升級系統

### 升級機制

英雄有 3 個等級。升級需要**經驗點（XP）**。

#### 升級路徑

```
1 級英雄 (起始)
    ↓
    需要 X 經驗點 (XP)
    ↓
2 級英雄 (更強大的卡牌)
    ↓
    需要 Y 經驗點 (XP)
    ↓
3 級英雄 (最強型態)
```

#### 獲取 XP 的方式

| 方式 | 獲得 XP |
|------|--------|
| 戰勝 2 級怪物 | 2 XP |
| 戰勝 3 級怪物 | 3 XP |
| 購買特定卡牌 | 0-2 XP（根據卡牌） |
| 使用特定村民卡 | 1-2 XP |

#### 升級的視覺與遊戲效果

當英雄升級：

1. **卡牌替換**：
   - 移除舊等級的英雄卡
   - 放入新等級的英雄卡
   - 新卡牌進入牌組

2. **能力提升**：
   ```
   例：塞維恩升級
   
   1 級：Attack +2
   2 級：Attack +3，Light +1
   3 級：Attack +3，Light +2，戰勝怪物時可購買光源物品
   ```

3. **戰術變化**：
   - 高等級英雄開啟新的牌組構築路線
   - 可能解鎖某些卡牌的購買權限

### 升級的戰略價值

**何時升級英雄？**

| 情境 | 建議 |
|------|------|
| 早期（回合 1-5） | 僅在有明確 XP 來源時升級 |
| 中期（回合 6-10） | 積極升級，為終局做準備 |
| 晚期（回合 11+） | 盡可能升級至 3 級，為最終戰鬥做準備 |

## 特殊機制

### 村莊破壞堆（Village Casualty Pile）

**機制**：

當怪物逃脫到村莊時，它被放入一個特殊的「破壞堆」。

| 事件 | 怪物進入破壞堆 |
|------|----------------|
| 選擇休養時，Rank 1 怪物逃脫 | ✓ |
| 玩家無法擊敗一隻怪物 | ✓ |
| 雷霆之石逃脫村莊外 | ✓（特殊） |

**遊戲終點影響**：

```
玩家最終分數 = 玩家卡牌 VP 總和 - 破壞堆怪物 VP 總和

勝利條件：
玩家最終分數 > 0 且 玩家控制雷霆之石
```

### 疾病卡（Disease）

**概念**：負面卡牌，降低牌組效率。

**來源**：
- 某些怪物戰鬥時強制獲得
- 例如「釋放的深淵怪」戰鬥時給予 1 疾病

**特性**：
- 疾病卡無 Gold、Attack、Magic Attack
- 佔用手牌空間，污染牌組
- 被特定英雄（聖杯系列）摧毀

**清除方式**：
1. **聖杯英雄** - 可摧毀疾病
2. **村民卡** - 解毒劑（摧毀疾病 + 抽牌）

### 等級懲罰（Level Penalty）

**概念**：低等級玩家對抗高等級怪物時的懲罰。

| 玩家等級 | 1 級怪物 | 2 級怪物 | 3 級怪物 |
|---------|---------|---------|---------|
| 1 級 | 無懲罰 | —— | —— |
| 2 級 | 無懲罰 | 無懲罰 | —— |
| 3 級 | 無懲罰 | 無懲罰 | 無懲罰 |

**—— 表示無法挑戰（如無引擎卡）**

## 遊戲結束與勝負判定

### 遊戲終止條件

遊戲在**以下任一情況**發生時結束：

| 條件 | 說明 |
|------|------|
| **1. 雷霆之石被擊敗** | 某位玩家成功戰勝持有雷霆之石的怪物 |
| **2. 雷霆之石逃脫** | 雷霆之石被推出地牢，進入破壞堆 |

### 終局計分（Final Scoring）

所有玩家在遊戲結束時進行計分：

#### 步驟 1：統計各自卡牌的勝利點

```
玩家 VP = Σ(所有卡牌上的 VP 值)
```

**包括**：
- 獲得的怪物卡（每隻怪物的 VP）
- 某些村莊卡（通常沒有 VP）

**例**：
```
玩家獲得卡牌：
- 青龍（VP 4）
- 白龍（VP 5）
- 水妖（VP 2）
- 其他怪物（VP 6）
- 英雄卡（VP 0）

總 VP = 4 + 5 + 2 + 6 = 17 VP
```

#### 步驟 2：統計破壞堆的怪物 VP

```
破壞堆 VP = Σ(逃脫怪物的 VP 值)
```

**例**：
```
破壞堆怪物：
- 怨恨獸（VP 2）
- 血顱獸人（VP 2）
- 奎力克哥布林（VP 3）

破壞堆 VP = 2 + 2 + 3 = 7 VP
```

#### 步驟 3：計算淨分

```
淨分 = 玩家 VP - 破壞堆 VP
```

**繼續例子**：
```
淨分 = 17 - 7 = 10 VP
```

### 勝利判定（多人模式）

**勝利條件（全部滿足）**：

1. ✓ **持有雷霆之石**：玩家卡牌中包含雷霆之石怪物
2. ✓ **淨分為正**：玩家 VP > 破壞堆 VP
3. ✓ **最高分**（如有多人贏）：分數最高者獲勝

**如果沒有玩家滿足上述所有條件**：
- 所有玩家都失敗
- 村莊被毀滅

### 勝利類型

| 勝利類型 | 條件 | 說明 |
|---------|------|------|
| **決定性勝利** | 1 位玩家滿足所有條件 | 該玩家獲勝 |
| **共同失敗** | 0 位玩家滿足條件 | 遊戲難度太高，所有玩家失敗 |

### 單人模式的勝敗

**單人模式勝利條件**：

```
✓ 玩家獲得雷霆之石怪物
✓ 玩家 VP ≥ 破壞堆 VP
```

**單人模式失敗條件**：

```
✗ 雷霆之石被完全推出地牢
✗ 無法阻止村莊被毀
```

## 平衡設計分析

### 資源流動平衡

```
┌──────────────────────────────────┐
│  資源轉換流 (Resource Loop)      │
├──────────────────────────────────┤
│  戰鬥獲勝 → 怪物卡 (VP + 金幣)   │
│     ↓                            │
│  金幣 → 購買村莊卡 (增強牌組)   │
│     ↓                            │
│  強化牌組 → 更容易戰勝 → 循環   │
└──────────────────────────────────┘
```

**設計精妙之處**：
- **正回饋迴圈**：越強越容易變得更強
- **負面困境**：牌組污染（疾病卡）限制無限增長
- **機會成本**：購物與戰鬥之間的平衡決策

### 英雄與英雄之間的平衡

| 英雄類型 | 攻擊力 | 特色 | 平衡手段 |
|---------|--------|------|---------|
| Fighter | 高 | 穩定傷害 | 無額外效果 |
| Archer | 中 | 卡牌抽取 | 需要配合 |
| Wizard | 中 | 法術引擎 | 需要法術支持 |
| Thief | 高 | 擾亂對手 | 單人無用 |
| Cleric | 中 | 疾病清除 | 特定場景有用 |

**設計原則**：
- 無絕對最強英雄
- 每個英雄有特定優勢場景
- 英雄升級後優勢更明顯

### 卡牌成本與效能的曲線

```
費用 vs 效能

│
│     ★ 火球/戰鎚（高成本高效能）
│    / \
│   /   \  火焰之劍（成本高，光源加成）
│  /     \
│ /       ★ 盛宴（中成本多效果）
│/         \
○──────────★─────────────
匕首      短劍      長柄武器
(低成本)  (低效能)  (中等)
```

**設計意圖**：
- 低費用卡牌快速強化初期牌組
- 高費用卡牌提供長期優勢
- 沒有絕對的「必買」卡牌

### 難度遞增的怪物曲線

```
怪物難度

等級 3  ★★★★★★★★★★  難度：極高
       ★★★★★★★★★★

等級 2  ★★★★★  難度：中等
           ★★★★

等級 1  ★★  難度：簡單
     ★★★

      早期    中期    終期
      ─────────────────→
```

**設計理由**：
- 玩家早期容易贏，建立信心
- 中期難度提升，需要牌組升級
- 終期面臨極大挑戰，需要完美牌組和策略

## 難度調整選項

### 難度預設

遊戲默認難度設定為「中等」。對於不同玩家群體，可進行調整。

### 簡單模式（Easy）

**調整**：

| 項目 | 修改 |
|------|------|
| 雷霆之石位置 | 牌庫底部 15 張內（而非 10 張） |
| 怪物 HP | -1（所有怪物） |
| 起始金幣 | +1 Gold（每回合購物時） |
| 破壞堆 VP | 不計入最終分數（或減半） |

**適用場景**：
- 新玩家教學
- 兒童友好版本
- 團隊希望輕鬆遊戲體驗

### 困難模式（Hard）

**調整**：

| 項目 | 修改 |
|------|------|
| 雷霆之石位置 | 牌庫底部 8 張內 |
| 怪物 HP | +1（所有怪物） |
| 怪物 VP | +1（逃脫時懲罰更大） |
| 起始卡牌 | 移除 1 張基本卡（初始更弱） |
| 英雄升級成本 | +X（升級需要更多 XP） |

**適用場景**：
- 經驗豐富的玩家
- 高難度挑戰愛好者
- 競技遊戲模式

### 傳奇難度（Legendary）

**調整**：

| 項目 | 修改 |
|------|------|
| 雷霆之石 | 必須在 Rank 1 被擊敗（無法遠距攻擊） |
| 怪物 HP | +2 |
| 破壞堆 VP | 翻倍計算 |
| 地牢大廳 | 7 個位置（而非 6 個） |
| 單人模式 | 怪物每回合自動推進 |

**適用場景**：
- 極端難度挑戰
- 速通競技（Speedrun）
- 單人模式最大挑戰
