## 1. 遊戲概述與單人規則

### 1.1 核心概念
玩家扮演一組冒險隊伍的領袖，透過購買裝備、雇用英雄、進入地城討伐怪物，最終目標是在怪物逃脫前擊敗持有雷霆之石的守護者（Boss）。

### 1.2 遊戲設置 (Setup)
* **村莊卡牌**：隨機選擇 4 種英雄、8 種村莊卡。
* **怪物牌庫**：
    1.  將 3 種怪物群洗勻。
    2.  取出 10 張牌面朝下疊好。
    3.  將「雷霆之石攜帶者」（BOSS）放在這 10 張牌上面。
    4.  將剩下的怪物牌疊在最上面（確保 BOSS 在後段出現）。
* **玩家起始牌庫**：6 張民兵 + 2 張匕首 + 2 張火把 + 2 張食物（標準配置）。

### 1.3 單人核心機制：地城行動與逃脫
在玩家回合結束時，根據執行的行動，地城會發生變化：

#### 情況 A：成功擊敗怪物
1.  將被擊敗的怪物收入棄牌堆。
2.  地城怪物向「等級 1」（出口）移動填補空缺。
3.  從怪物牌庫翻開新卡填滿深處。
4.  **結果：無怪物逃脫。**

#### 情況 B：前往村莊 / 休整 / 戰敗
1.  **怪物逃脫**：位於地城「等級 1」的怪物成功逃脫（移出遊戲）。
2.  **推進**：剩下的怪物向出口推進一格。
3.  **補充**：翻開一張新卡填補深處。
4.  **判負條件**：若逃脫的是 BOSS，遊戲直接失敗。

## 2. 遊戲設計企劃書 (Game Design Document)

### 2.1 遊戲物件定義 (Game Objects)

#### 2.1.1 卡牌屬性
所有卡牌繼承以下屬性：
* **Cost**：購買花費。
* **Gold**：提供的金幣值。
* **VP**：勝利點數。
* **Type**：英雄 / 武器 / 物品 / 法術 / 怪物。

**英雄 (Hero) 特有屬性：**
* **Level**：等級 (1-3)。
* **Strength**：力量 (負重上限)。
* **Attack**：物理/魔法攻擊力。
* **XP_Cost**：升級所需經驗。

**地城環境 (Dungeon Slots)：**
* **Rank 1 (出口)**：光照懲罰 -1
* **Rank 2**：光照懲罰 -2
* **Rank 3**：光照懲罰 -3
* **Rank 4 (深處)**：光照懲罰 -4 (怪物生成點)

### 2.2 核心迴圈 (Core Loop)
```text
Start Turn -> 抽牌(6張)
    |
    +-> [決策分支]
    |     A. 村莊行動 (購買 / 升級) -> 觸發怪物逃脫
    |     B. 休整行動 (調整手牌)   -> 觸發怪物逃脫
    |     C. 地城行動 (戰鬥)
    |           |-> 勝利 -> 怪物被消滅，不觸發逃脫
    |           |-> 失敗 -> 觸發怪物逃脫
    |
    +-> 地城更新 (Shift & Spawn)
    +-> 檢查勝敗條件
    +-> 清理與補牌 -> Next Turn
    
2.3 戰鬥計算公式 (Battle Logic)
程式實作邏輯如下：

計算總光照 (Total Light) Total_Light = Item_Light + Spell_Light + Hero_Ability_Light

計算光照懲罰 (Light Penalty) Penalty_Factor = Max(0, Dungeon_Rank_Darkness - Total_Light) Attack_Penalty = Penalty_Factor * 2 (通常每缺1光照扣2攻)

計算最終攻擊力 (Final Attack) Raw_Attack = Sum(Hero_Atk + Weapon_Atk) Final_Attack = Raw_Attack - Attack_Penalty

勝負判定 IF (Final_Attack >= Monster_Health) THEN Win ELSE Lose
